Dropdown Menus: An Accessibility Evaluation

Project Focus

The case study below focuses on the accessibility of dropdown menus, aiming to improve them for all users, including those using keyboard, mouse, touchscreen, or screen readers.

By evaluating—and potentially reworking—current implementations, the goal is to enhance efficiency and accessibility of dropdowns without sacrificing their functionality.

Current Implementation

Analysis of Pinterest's existing dropdown menu system revealed several areas for improvement:

Keyboard Navigation

Limited keyboard support for navigating between boards and sections

Visual Feedback

Unclear distinction between selected and unselected states

Navigation Flow

Inefficient process for switching between boards

Learnability

Users must remember previous selections without clear visual cues

Component Redesign

Pinterest's redesigned board and section selection dropdown. The new layout is more intuitive, with a clear visual distinction between selected and unselected states, and a more efficient navigation flow.

Key Improvements

  • Side-by-Side Layout: The section list now opens to the right of the board list, making it easier for users to understand which board they've chosen
  • Improved Efficiency: Switching to a different board (or displaying its sections) only requires clicking on it within the left list, instead of navigating back to the all-boards list
  • Visual Clarity: Previously chosen board has a light grey background, and a hovered-over board/section has a medium gray background
  • Enhanced Learnability: No need to remember the previous selection—it's always visible
  • Keyboard Navigation: Users can now navigate through sections with up/down arrows, open/close sections with right/left arrows, or progress beyond the chosen board's sections into those of the subsequent board in the main list with tab

Reflection

This redesign prioritizes desktop users (whether keyboard or mouse), as the side-by-side layout couldn't be implemented on mobile. Future iterations would explore mobile-optimized solutions that maintain the improved accessibility and efficiency.

The focus on keyboard navigation and clear visual feedback demonstrates how thoughtful component design can significantly improve the experience for users with diverse needs and interaction preferences.
